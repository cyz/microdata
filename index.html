O que são os Microdados?

São um subconjunto de informações em um documento que tem um significado para máquinas e robôs, assim como tem um significado para uma pessoa lendo um documento.

Por que utilizar microdados?

De forma geral, desenvolvedores dedicam-se mais ao estudo/desenvolvimento de outras funcionalidades conquistas através do HTML5, pórem quando o assunto é SEO, o uso de microdados é um grande aliado.

Utilizando Microdata você pode ajudar motores de busca e outras aplicações a entender melhor o seu conteúdo e apresentá-lo de forma útil e relevante.
Ao adicionar microdados na estrutura HTML de suas páginas, elas "dizem":
"Hey motor de busca, esta informação descreve este filme específico".

Como eu faço?

No site schema.org, você encontra uma lista completa de todos os tipos de esquemas (entidades) que já foram definidas.

Os vocabulários pertencem a uma hierarquia, exatamente como se fosse uma hierarquia de classes em OO, em que cada classe pode estender outra classe, herdando assim todas as propriedades e comportamentos da classe pai. No schema.org temos apenas 2 tipos básicos de entidades: DataType e Thing.

O que é DataType?

Este atributo define vários tipos de dados primitivos que servem para descrever os tipos de dados que serão usados para preencher um atributo qualquer em qualquer vocabulário.

O que é Thing?

É a entidade mais abstrata a nível semântico. Thing pode significar qualquer coisa. As 7 entidades que a estendem são: CreativeWork, Event, Intangible, Organization, Person, Place e Product.

Propriedades microdata

Microdata define 5 diferentes atributos que podem ser adicionados a qualquer elemento HTML o qual você queira descrever semanticamente. Estes atributos são:

  • itemscope: Indica que o item é uma entidade semântica e que todas as propriedades que estiverem dentro dele contêm informações sobre ele;
  • itemtype: Indica a entidade (o tipo). Aqui você deve especificar a URL do vocabulário (por exemplo: http://schema.org/CreativeWork);
  • itemid: Um identificador único para o item (leia itemref);
  • itemprop: Indica que o item possui o valor da propriedade especificada (veja o exemplo abaixo);
  • itemref: Propriedades que estão em elementos fora do escopo (elemento com itemscope) podem ser associadas a este item. itemref deve conter seu valor preenchido com o itemid do elemento o qual se deseja fazer a associação.

Exemplos

Breadcrumb

		
<div itemscope itemtype="http://data-vocabulary.org/Breadcrumb">
	<a href="http://www.meusite.com.br" itemprop="url">
		<span itemprop="title">Home</span>
	</a>
</div>  
<div itemscope itemtype="http://data-vocabulary.org/Breadcrumb">
	<a href="http://www.meusite.com.br/contato.html" itemprop="url">
		<span itemprop="title">Contato</span>
	</a>
</div>

Produto

		
<div itemscope itemtype="http://schema.org/Product">
	<img itemprop="image" src="dell-30in-lcd.jpg" />
	<span itemprop="name">Dell UltraSharp 30" LCD Monitor</span>

	<div itemprop="aggregateRating" itemscope itemtype="http://schema.org/AggregateRating">
			<span itemprop="ratingValue">87</span>
			out of <span itemprop="bestRating">100</span>
			based on <span itemprop="ratingCount">24</span> user ratings
	</div>

	<div itemprop="offers" itemscope itemtype="http://schema.org/AggregateOffer">
		<span itemprop="lowPrice">$1250</span>
		to <span itemprop="highPrice">$1495</span>
		from <span itemprop="offerCount">8</span> sellers
	</div>
</div>
